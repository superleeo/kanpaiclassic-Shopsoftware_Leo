## 🚀 系统安装

### 安装前检查清单

```
□ PHP 7.4+ 已安装
□ MySQL 5.7+ 或 MariaDB 已安装并运行
□ 数据库用户和密码已准备
□ Web 服务器 (Apache/Nginx) 已配置
□ 文件上传限制设置为至少 50MB
□ PHP 内存限制设置为至少 256MB
□ 所有必要目录具有写入权限
```

### 安装步骤

**Step 1: 访问安装程序**
```
1. 浏览器打开: http://yourdomain.com/admin/install/install.php
2. 选择语言 (Deutsch / English / 中文)
3. 阅读许可条款
```

**Step 2: 数据库配置**
```
输入以下信息：
- 数据库主机: localhost (或数据库服务器IP)
- 数据库端口: 3306 (默认)
- 数据库用户: root 或其他用户
- 数据库密码: [您的密码]
- 数据库名称: flow_shop_miaowei (建议名称)

点击 "继续" 进行连接测试
```

**Step 3: 表前缀配置**
```
建议使用: flow_shop_
这样所有表将命名为:
- flow_shop_artikel (商品表)
- flow_shop_kategorien (分类表)
- 等等...
```

**Step 4: 创建表和导入数据**
```
系统将自动执行 install.sql
- 创建所有需要的表
- 设置关键和索引
- 创建默认数据
```

**Step 5: 管理员账户设置**
```
设置您的管理员账户：
- 用户名: [选择一个强用户名，不要用 admin]
- 邮箱: info@miaowei-teppanyaki.de
- 密码: [设置一个强密码，至少 12 字符]
- 密码确认: [重复输入]

记住: 将凭据安全保存！
```

**Step 6: 店铺基本信息**
```
填入店铺信息：
- 店铺名称: 妙味日本料理铁板烧
- 地址: Augustaanlage 15
- 城市: Mannheim
- 邮编: 68161
- 国家: Deutschland
- 电话: +49 621 XXXXXX
- 邮箱: info@miaowei-teppanyaki.de
- 税号: DE123456789
```

**Step 7: 安装完成**
```
✅ 安装成功
- config.inc.php 已创建在 admin/ 目录
- 系统现在已准备好使用
- 删除或重命名 install.php 文件以增强安全性
```

## 📋 订单处理

### 订单工作流

**查看订单**: 销售 > 订单管理

```
订单列表显示：
- 订单号
- 客户名字
- 订单日期
- 金额
- 状态
```

### 订单状态管理

```
默认订单状态：
1. 待确认 (Neu)
2. 已确认 (Bestätigt)
3. 配送中 (Versand)
4. 已送达 (Geliefert)
5. 已取消 (Storniert)
6. 退货 (Rückgabe)

处理流程：
1. 新订单到达 → 立即查看
2. 确认订单 → 更新状态为 "已确认"
3. 打包和配送 → 更新状态为 "配送中" + 添加跟踪号
4. 确认送达 → 更新状态为 "已送达"
5. 记录反馈 → 如有问题添加备注
```

### 订单详情查看

```
点击订单号查看完整详情：
- 客户信息
- 配送地址
- 账单地址
- 商品清单（名称、数量、价格）
- 运费和税收明细
- 支付方式和状态
- 订单备注和历史
```

### 处理问题订单

#### 订单被拒收

```
1. 致电客户确认地址
2. 重新安排配送
3. 或提供退款选项
4. 更新订单状态为 "重新安排"
5. 记录所有通讯
```

#### 客户投诉质量

```
1. 即时查看商品照片（如提供）
2. 核实库存中该商品的状况
3. 提供解决方案：
   - 补送商品
   - 部分退款
   - 全额退款
4. 记录问题以改进流程
```

#### 支付失败

```
1. 检查支付网关状态
2. 通知客户重试支付
3. 设置支付提醒通知
4. 如果多次失败，取消订单并通知
```

## ❓ 常见问题

### Q1: 如何处理缺货情况？

```
A: 在商品编辑页面：
1. 设置库存数量为 0
2. 启用 "缺货时隐藏" 选项
3. 或启用 "预订" 功能
4. 前台会显示 "暂时缺货"
5. 当库存恢复时重新上架
```

### Q2: 如何修改已发送的订单确认邮件？

```
A: 
1. 系统设置 > 邮件模板
2. 找到 "订单确认" 模板
3. 编辑 HTML 内容
4. 保存
5. 新的邮件模板将用于下一个订单
```

### Q3: 如何为不同客户设置不同价格？

```
A: 
1. 创建会员等级 (参见营销部分)
2. 为不同等级设置折扣
3. 或在 商品编辑 > 分级价格
4. 为特定客户分配特殊价格
```

### Q4: 如何导出销售报告？

```
A:
1. 系统设置 > 报告
2. 选择报告类型 (按日期、按产品等)
3. 设置时间范围
4. 点击 "导出"
5. 下载 CSV 或 PDF 格式
```

### Q5: 客户忘记了账户密码怎么办？

```
A:
1. 在登录页面点击 "忘记密码"
2. 输入邮箱地址
3. 系统发送重置链接
4. 客户点击链接设置新密码

如果客户无法访问邮箱：
1. 管理后台 > 客户管理
2. 找到客户账户
3. 点击 "重置密码"
4. 设置临时密码
5. 发送给客户
```

### Q6: 系统出错怎么办？

```
A:
1. 检查错误日志：
   - 系统设置 > 错误日志
   - 查看最近的错误

2. 常见错误排除：
   - 数据库连接失败 → 检查数据库服务
   - 上传文件失败 → 检查目录权限
   - 邮件发送失败 → 检查 SMTP 配置
   - 显示乱码 → 检查字符编码设置

3. 如果无法解决：
   - 查看日志文件: /admin/logs/
   - 运行系统诊断工具
   - 与技术支持联系
```
